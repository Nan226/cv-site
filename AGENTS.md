# CV-Site 协作规则

本文件用于 Codex、Claude Code、VS Code 内的 AI 助手及人工开发者之间交接项目。

## 沟通与执行

- 与用户沟通、编写实施计划和汇报改动时使用中文。
- 开始工作前先阅读本文件、`Logs/` 中最新的日志，以及与任务相关的现有代码。
- 用户要求分步骤推进。每完成一个主要步骤后先汇报结果并等待用户确认，不要一次完成所有后续步骤。
- 对需求存在关键歧义时先询问用户；可以从现有文件或图片中确认的信息，不重复询问。

## 日志要求

- 每次进行文件创建、修改、删除、命令执行、设计决策或验证，都要记录到 `Logs/`。
- 日志文件按日期命名：`Logs/YYYY-MM-DD.md`；当天文件存在时追加，不重复创建。
- 日志至少写明：时间或日期、执行者、操作、涉及文件、结果，以及仍待确认的事项。
- 开始新会话时先读最新日志；结束一个步骤前先补全日志。
- 聊天上下文不视为项目记录。重要需求、决定和进度必须落盘到日志或本文件。

## 多助手协作

- Codex 与 Claude Code 不共享聊天会话，只通过工作区文件、日志和版本控制状态交接。
- 不得假设自己知道另一个助手尚未写入文件的计划或推理。
- 修改前检查现有文件，不覆盖或回退用户及其他助手的改动。
- 发现不属于当前任务的改动时保留原状；若与当前任务冲突，先记录并向用户说明。
- 交接时在当天日志中注明已完成内容、未完成内容、下一步和已知风险。

## 当前产品约束

- 当前阶段只制作个人简历网站的 HOME 页面。
- 技术栈为纯 HTML、CSS 和原生 JavaScript，除非用户明确同意更换。
- 页面需适配窗口尺寸，目标是单屏展示且不出现页面滚动。
- 视觉以 `images/Home参考.png` 和 `images/HOME/` 下的细节参考图为准。
- 页面整体以白色为主，融合非常淡的蓝色、粉色或紫粉色光晕，避免明显分隔色条。
- 主标题为 `YE NAN`，副标题为 `Builder · Dreamer · Braver`。
- 导航为 `About Me`、`Internship`、`Projects`、`Skills`、`Learning`。
- 人物左右各排列五个标签；人物后续需要轻微 3D 鼠标跟随和分部位点击反馈。
- 导航标签后续需要实现撕裂或撕碎悬浮效果。

## 修改与验证

- 保持改动范围聚焦当前步骤，不顺手重构无关内容。
- 图片素材默认不改动；如需裁切、压缩或生成新素材，先说明用途并保留原文件。
- 页面样式修改后必须在桌面尺寸和窄屏尺寸检查布局，确保文字、人物、标签互不遮挡。
- 完成验证后把验证方式和结果写入当天日志。
