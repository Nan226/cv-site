## 2. 页面目标

About Me 页面用于呈现 Ye Nan 的成长路径、教育背景、城市迁移和关键人生阶段。

页面不应被制作成传统的长篇自我介绍，而应通过一条清晰的时间线和可交互的经历卡片，让访问者在较短时间内理解：

1. Ye Nan 从哪里出发。
2. 经历过哪些重要阶段。
3. 每段经历带来了什么能力或认知变化。
4. 现在正在走向哪里。

页面整体需要延续 HOME 的个人品牌感：轻盈、聪明、有创造力，同时保持简历网站需要的信息清晰度。

## 3. 用户与使用场景

### 3.1 主要用户

- 招聘方与面试官。
- 潜在合作伙伴。
- 对 Ye Nan 的项目、成长经历和个人方向感兴趣的访问者。

### 3.2 核心使用场景

- 用户从 HOME 点击 `About Me` 导航进入该页面。
- 用户快速浏览时间节点，了解成长路径。
- 用户拖动、滚动或点击卡片，查看某个阶段的详细信息。
- 用户通过卡片中的图片和关键词建立对个人经历的直观记忆。

## 4. 产品范围

### 4.1 本阶段包含

- About Me 页面完整视觉结构。
- 页面标题和一句简短的个人介绍。
- 横向成长时间线。
- 6 个时间节点。
- 与时间节点对应的 6 张经历卡片。
- 卡片环绕滚动或横向透视轮播。
- 桌面端和窄屏端适配。
- 鼠标、触控和键盘基础交互。
- 与 HOME 页面之间的导航衔接。

### 4.2 本阶段不包含

- Internship、Projects、Skills、Learning 页面。
- 后台内容管理系统。
- 用户登录、评论、分享或收藏。
- 自动获取社交平台数据。
- 复杂 WebGL 3D 卡片场景。
- 对原始照片进行不可逆修改。

## 5. 信息架构

About Me 页面由四部分组成：

1. **页面导航**
   - 延续 HOME 的顶部导航。
   - `About Me` 显示为当前选中状态。

2. **页面标题区**
   - 主标题：`About Me`
   - 副标题：一句描述成长状态的英文短句。
   - 建议文案：`Still becoming, always building.`
   - 最终文案待用户确认。

3. **成长时间线**
   - 横向展示 6 个关键节点。
   - 每个节点显示日期、城市和简短阶段名称。
   - 当前激活节点需要有明确视觉反馈。

4. **经历卡片轮播**
   - 每个节点对应一张卡片。
   - 中间卡片为当前激活卡片。
   - 两侧卡片缩小、降低对比度并产生轻微透视。
   - 用户切换卡片时，时间线同步切换。

### 6.3 卡片内容模板

每张卡片统一包含：

| 内容字段 | 说明 |
|---|---|
| `type` | 阶段类型，例如 Education、Internship、Growth、Life |
| `date` | 时间范围 |
| `location` | 城市或学校所在地 |
| `title` | 学校、公司、项目或人生阶段名称 |
| `subtitle` | 专业、岗位或阶段身份 |
| `description` | 1 至 2 句经历说明 |
| `keywords` | 2 至 4 个关键词 |
| `image` | 对应照片、证件、场景或插画 |
| `accent` | 卡片强调色 |

### 6.4 卡片文案原则

- 每张卡片只表达一个阶段主题。
- 标题应直接说明学校、组织、项目或阶段，不使用空泛口号。
- 描述控制在 45 至 80 个中文字符，或 25 至 45 个英文单词。
- 关键词应体现该阶段获得的能力、视角或变化。
- 不重复 Internship 和 Projects 页面未来会详细展开的职责与成果。
- About Me 更侧重“为什么变化”和“如何成长”，其他页面再讲具体工作结果。

## 7. 视觉设计要求

### 7.1 整体风格

- 延续 HOME 的白色主背景。
- 使用非常淡的蓝色、粉色和紫粉色光晕自然融合。
- 不使用明显分隔的色条或大面积纯紫背景。
- 页面需要轻盈、干净，但不能显得像空白模板。
- 视觉重点依次为：标题、当前时间节点、当前经历卡片。

### 7.2 色彩建议

| 用途 | 建议颜色 |
|---|---|
| 主文字 | `#20222A` |
| 次级文字 | `#6B6E7A` |
| 蓝色强调 | `#79B8FF` |
| 粉色强调 | `#FF9CCB` |
| 紫粉过渡 | `#B69CFF` |
| 时间线默认 | `rgba(126, 112, 180, 0.25)` |
| 卡片背景 | `rgba(255, 255, 255, 0.82)` |
| 卡片边框 | `rgba(125, 112, 170, 0.14)` |

实际开发时应复用 HOME 已存在的 CSS 变量，避免形成第二套互不一致的颜色系统。

### 7.3 字体层级

- 页面主标题使用与 `YE NAN` 相同或相近的展示字体气质。
- 主标题可使用蓝粉渐变，但字号应小于 HOME 主标题。
- 时间、地点和分类使用紧凑的无衬线字体。
- 卡片正文以可读性优先，不使用过细字重。
- 所有字距保持为 `0`，不使用负字距。

### 7.4 卡片外观

- 卡片圆角不超过 `8px`。
- 卡片以图片为主要视觉，不制作成大段文字面板。
- 当前卡片可有轻微阴影和柔和光晕。
- 两侧卡片通过缩放、透明度和 Y 轴旋转表现纵深。
- 不在卡片内部再嵌套另一张完整卡片。

## 8. 布局规格

### 8.1 桌面端

- 页面最小目标视口：`1280 x 720`。
- 页面可使用单屏沉浸展示，主体内容在视口内完成浏览。
- 顶部导航高度沿用 HOME。
- 标题区位于导航下方，占页面高度约 18%。
- 时间线位于页面中部，占页面高度约 16%。
- 卡片轮播位于页面下半部分，占页面高度约 52%。
- 页面底部保留必要安全间距。

推荐布局比例：

| 区域 | 建议高度 |
|---|---|
| 导航 | 6% 至 8% |
| 标题区 | 15% 至 20% |
| 时间线 | 14% 至 18% |
| 卡片区 | 48% 至 56% |

### 8.2 窄屏端

- 时间线允许横向滚动，但隐藏默认滚动条。
- 当前节点自动滚动至可视区域中心。
- 卡片轮播每次主要展示一张卡片，并露出下一张卡片边缘。
- 减弱卡片透视角度，避免文字因旋转而难以阅读。
- 标题、副标题、时间线和卡片之间不得重叠。
- 触控滑动距离应适中，防止轻微页面滚动被误判为切换卡片。

## 9. 交互设计

### 9.1 页面进入

- 从 HOME 点击 `About Me` 后平滑移动到 About 页面。
- 页面进入后标题轻微淡入上移。
- 时间线从左向右展开。
- 当前卡片在时间线之后出现。
- 总入场时长建议控制在 700 至 1100 毫秒。

### 9.2 时间线交互

- 点击任意节点，切换到对应卡片。
- 激活节点使用蓝粉渐变圆点或柔和光晕。
- 已经过的节点可使用较深颜色。
- 未激活节点保持低对比度。
- 节点切换后，时间、地点和卡片内容同步更新。

### 9.3 卡片轮播交互

桌面端支持：

- 鼠标滚轮横向切换。
- 鼠标按住拖动切换。
- 点击左右卡片切换。
- 点击左右箭头切换。
- 键盘方向键切换。

移动端支持：

- 手指左右滑动切换。
- 点击时间线节点切换。
- 点击分页圆点或箭头切换。

### 9.4 环绕视觉逻辑

卡片位置以当前卡片索引为中心计算：

- 当前卡片：`scale(1)`，透明度 `1`，Y 轴旋转接近 `0deg`。
- 相邻卡片：`scale(0.82-0.9)`，透明度 `0.65-0.82`，Y 轴旋转约 `±18deg`。
- 更远卡片：缩小并降低透明度，可只露出边缘。
- 首尾卡片可循环衔接，形成环绕感。

循环滚动必须保持索引稳定，不能在首尾切换时产生明显跳帧。

### 9.5 卡片悬浮

- 当前卡片悬浮时仅轻微上移或增强阴影。
- 非当前卡片悬浮时提示可点击，但不能抢过当前卡片的视觉权重。
- 禁止大幅弹跳、过度倾斜或持续晃动。

### 9.6 动效降级

当系统启用 `prefers-reduced-motion: reduce` 时：

- 关闭透视旋转和大幅位移动画。
- 保留即时卡片切换和清晰的选中状态。
- 禁止自动播放轮播。

## 10. 导航与页面关系

- HOME 的 `About Me` 导航链接进入 `#about`。
- About 页面导航中的 `About Me` 需要显示选中状态。
- 用户可通过滚动或返回 HOME 控件回到首页。
- 当前阶段不为未完成的其他导航页面制造空白跳转。
- 后续新增其他页面时，About 页面组件不得阻断纵向页面导航。

## 11. 内容数据结构建议

经历内容建议在 JavaScript 中使用结构化数组维护：

```js
const aboutTimeline = [
  {
    date: "2003.02",
    location: "Quanzhou, Fujian",
    type: "Life",
    title: "To be confirmed",
    subtitle: "To be confirmed",
    description: "To be confirmed",
    keywords: ["To be confirmed"],
    image: "images/about/example.jpg",
    accent: "#79B8FF"
  }
];
```

要求：

- 页面结构与内容数据分离。
- 不把 6 张卡片的切换逻辑写成 6 组重复代码。
- 图片必须提供有意义的 `alt` 文本。
- 内容缺失时显示开发占位，不在正式页面展示 `To be confirmed`。

## 12. 状态设计

### 12.1 默认状态

- 默认激活第一个节点，或用户最希望突出展示的节点。
- 第一张卡片位于视觉中心。

### 12.2 图片加载状态

- 图片加载前显示与卡片色调一致的浅色占位。
- 图片加载失败时保留卡片文字，不允许卡片塌陷。

### 12.3 内容缺失状态

- 开发阶段允许使用统一占位图和占位文字。
- 正式发布前，所有节点必须具有标题、描述和图片。

### 12.4 边界状态

- 快速连续切换时只响应最终目标索引。
- 拖动距离不足时卡片回弹到原位。
- 页面失去焦点时停止非必要动画。

## 13. 无障碍要求

- 时间线使用可聚焦按钮，不使用无法操作的纯 `div`。
- 当前节点提供 `aria-current="step"`。
- 轮播区域提供清晰的可访问名称。
- 卡片切换后通过适度的 `aria-live` 告知当前阶段。
- 所有交互均可使用键盘完成。
- 正文和背景对比度满足 WCAG AA。
- 焦点样式清晰可见，不能只依赖颜色变化。

## 14. 性能要求

- About 页面不新增大型 3D 模型。
- 照片使用 WebP 或经过压缩的 JPEG/PNG。
- 单张卡片主图建议不超过 300 KB。
- 非当前卡片图片可延迟加载。
- 动画优先使用 `transform` 和 `opacity`。
- 避免在鼠标移动时频繁修改布局属性。
- 页面切换和轮播目标保持接近 60 FPS。

## 15. 响应式断点建议

| 断点 | 行为 |
|---|---|
| `>= 1200px` | 完整时间线和 5 张可见层叠卡片 |
| `768px - 1199px` | 缩小卡片间距，显示 3 张主要卡片 |
| `< 768px` | 单卡片为主，时间线横向滚动 |
| `< 480px` | 精简辅助文案，确保标题和卡片信息完整可读 |

断点应根据实际内容适配，不应仅按设备名称判断。

## 16. 数据埋点建议

当前静态版本可以暂不接入分析服务，但建议预留以下事件名称：

- `about_view`
- `timeline_node_click`
- `about_card_swipe`
- `about_card_arrow_click`
- `about_card_keyboard_change`

未来接入分析工具时，不应采集敏感个人信息。

## 17. 验收标准

### 17.1 内容验收

- 6 个时间节点的日期、地点和经历准确。
- 每个节点都有对应卡片。
- 卡片标题、正文、关键词和图片均已确认。
- 页面不存在正式发布用的占位文案。

### 17.2 视觉验收

- About 页面与 HOME 的蓝粉色视觉保持一致。
- 标题、时间线和卡片层级明确。
- 环绕卡片有纵深感，但当前卡片内容清晰可读。
- 页面不存在明显色条、嵌套卡片和过度紫色化。
- 桌面端与窄屏端无文字、卡片和导航重叠。

### 17.3 交互验收

- 点击时间线节点可准确切换卡片。
- 点击、拖动、滚轮、触控和键盘切换行为稳定。
- 时间线与卡片的激活状态始终一致。
- 首尾循环时没有明显跳动或索引错误。
- 快速连续操作不会导致页面卡死或卡片错位。

### 17.4 性能验收

- 页面首次进入无明显长时间白屏。
- 图片加载失败不会破坏布局。
- 常见桌面浏览器中动画流畅。
- 窄屏设备上滑动操作不与页面导航严重冲突。

## 18. 开发拆分建议

### 第一步：静态结构

- 创建 About 标题区。
- 创建时间线。
- 创建 6 张静态卡片。
- 完成桌面端基础布局。

### 第二步：轮播交互

- 建立统一数据结构。
- 实现节点与卡片双向同步。
- 实现点击、箭头和键盘切换。

### 第三步：环绕动效

- 增加卡片透视、缩放和层级。
- 增加拖动、滚轮与触控滑动。
- 完成首尾循环。

### 第四步：响应式与质量检查

- 完成窄屏布局。
- 增加无障碍状态。
- 压缩图片并验证加载性能。
- 在桌面和移动视口完成视觉验收。

每完成一个主要步骤后，应先向用户展示效果并等待确认，再进入下一步。

## 19. 待用户确认清单

1. About 页面是否继续使用 HOME 的蓝粉渐变，而不是原参考图的紫色主调。
2. 页面副标题是否使用 `Still becoming, always building.`。
3. 6 个时间节点的准确日期和地点。
4. 每个节点对应的真实经历、学校、公司或项目。
5. 每张卡片需要使用的照片。
6. 默认突出展示第一段经历，还是最近的一段经历。
7. 卡片是否允许无限循环。
8. About 页面是否保持单屏，还是允许卡片内容较多时纵向滚动。

## 20. 当前推荐决策

- 视觉继续沿用 HOME 的白底蓝粉渐变。
- 桌面端优先保持单屏展示。
- 默认激活第一节点，形成从过去到未来的叙事顺序。
- 卡片允许无限循环，但时间线仍按照真实时间从左到右排列。
- 第一版先完成点击和箭头切换，再增加拖动与触控，降低一次实现的风险。
- 在用户补充真实履历前，只搭建结构，不发布虚构内容。
